# ESLint Extended Config - Quick Reference

## Quick Commands

```bash
# Lint a file
npx eslint src/file.ts --config eslint.config.extended.mjs

# Lint and auto-fix
npx eslint src --config eslint.config.extended.mjs --fix

# Lint with cache (faster)
npx eslint src --config eslint.config.extended.mjs --fix --cache

# Lint specific module
npx eslint src/modules/user --config eslint.config.extended.mjs --fix

# Check without auto-fix (CI)
npx eslint src --config eslint.config.extended.mjs --max-warnings 0
```

## Most Common Fixes

### 1. No Any Types

```typescript
// ❌ Bad
function process(data: any): any { return data; }

// ✅ Good
function process(data: ProcessInput): ProcessOutput { return data; }
function process(data: unknown): ProcessOutput { /* type guard */ }
```

### 2. Explicit Return Types

```typescript
// ❌ Bad
async function getUser(id: string) { }

// ✅ Good
async function getUser(id: string): Promise<User | null> { }
```

### 3. No Console

```typescript
// ❌ Bad
console.log('message');

// ✅ Good
private readonly logger = new Logger(ClassName.name);
this.logger.log('message');
```

### 4. Prefer Const

```typescript
// ❌ Bad
let user = await findUser(id);
return user;

// ✅ Good
const user = await findUser(id);
return user;
```

### 5. Remove Unused Imports

```typescript
// ❌ Bad
import { Injectable, Logger } from '@nestjs/common';
class Service { } // Logger unused

// ✅ Good
import { Injectable } from '@nestjs/common';
class Service { }
```

### 6. Naming Conventions

```typescript
// ❌ Bad
interface IUser { }
enum status { }
const Max_Retries = 3;

// ✅ Good
interface User { }
enum UserStatus { }
const MAX_RETRIES = 3;
```

### 7. Class Suffixes (NestJS)

```typescript
// ❌ Bad
class UserHandler { }

// ✅ Good
class UserService { }
class UserController { }
class UserGuard { }
class UserDto { }
```

### 8. Accessibility Modifiers

```typescript
// ❌ Bad
@Injectable()
class UserService {
  userRepo: Repository<User>;
  getUser() { }
}

// ✅ Good
@Injectable()
class UserService {
  private readonly userRepo: Repository<User>;
  public async getUser(): Promise<User> { }
}
```

### 9. Strict Boolean Expressions

```typescript
// ❌ Bad
if (user) { }

// ✅ Good
if (user !== null) { }
if (user !== undefined) { }
```

### 10. Nullish Coalescing

```typescript
// ❌ Bad
const name = user.name || 'default';

// ✅ Good
const name = user.name ?? 'default';
```

## NestJS Patterns

### Service
```typescript
import { Injectable, Logger } from '@nestjs/common';
import type { Repository } from 'typeorm';
import type { User } from './user.entity';

@Injectable()
export class UserService {
  private readonly logger = new Logger(UserService.name);

  constructor(
    private readonly userRepository: Repository<User>,
  ) {}

  public async findAll(): Promise<User[]> {
    return this.userRepository.find();
  }

  private validateUser(user: User): boolean {
    return user.email !== null;
  }
}
```

### Controller
```typescript
import { Controller, Get, Post, Body, Param } from '@nestjs/common';
import type { User } from './user.entity';
import { UserService } from './user.service';
import { CreateUserDto } from './dto/create-user.dto';

@Controller('users')
export class UserController {
  constructor(private readonly userService: UserService) {}

  @Get()
  public async findAll(): Promise<User[]> {
    return this.userService.findAll();
  }

  @Post()
  public async create(@Body() dto: CreateUserDto): Promise<User> {
    return this.userService.create(dto);
  }
}
```

### DTO
```typescript
import { IsEmail, IsString, MinLength } from 'class-validator';

export class CreateUserDto {
  @IsEmail()
  email: string;

  @IsString()
  @MinLength(2)
  name: string;
}
```

### Entity
```typescript
import { Entity, PrimaryGeneratedColumn, Column } from 'typeorm';

@Entity()
export class User {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column()
  email: string;

  @Column()
  name: string;
}
```

## Type Imports

```typescript
// ✅ Separate type imports
import type { User } from './user.entity';
import { CreateUserDto } from './dto/create-user.dto';

// ✅ Inline type imports
import { type User, CreateUserDto } from './user';
```

## Disabling Rules

### Single Line
```typescript
// eslint-disable-next-line @typescript-eslint/no-explicit-any
function legacy(data: any): any { return data; }
```

### Multiple Lines
```typescript
/* eslint-disable @typescript-eslint/no-explicit-any */
function legacy1(data: any): any { return data; }
function legacy2(data: any): any { return data; }
/* eslint-enable @typescript-eslint/no-explicit-any */
```

### Entire File
```typescript
/* eslint-disable @typescript-eslint/no-explicit-any */
// Legacy file
```

## File Overrides

These files have relaxed rules:

- `*.spec.ts` - Tests (any allowed, no return types)
- `*.dto.ts` - DTOs (no return types, no public modifiers)
- `*.entity.ts` - Entities (no public modifiers)
- `*.migration.ts` - Migrations (any allowed, console allowed)
- Config files - (any allowed, console allowed)

## Common Error Messages

| Error | Fix |
|-------|-----|
| `Missing return type on function` | Add `: ReturnType` after parameters |
| `Unexpected any value` | Replace `any` with specific type or `unknown` |
| `Unsafe member access .X on an any value` | Add proper types instead of `any` |
| `Prefer using nullish coalescing` | Replace `\|\|` with `??` |
| `Unexpected nullable X value in conditional` | Use `!== null`, `!== undefined` |
| `Unsafe call of an any typed value` | Add proper function types |
| `Missing accessibility modifier` | Add `public`, `private`, or `protected` |

## Quick Type Fixes

### Unknown to Specific Type
```typescript
function process(data: unknown): Result {
  // Type guard
  if (typeof data === 'object' && data !== null && 'id' in data) {
    const typed = data as { id: string };
    return processTyped(typed);
  }
  throw new Error('Invalid data');
}
```

### Any to Generic
```typescript
// ❌ Before
function first(arr: any[]): any {
  return arr[0];
}

// ✅ After
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}
```

### Infer Return Type
```typescript
// Hover over function in VSCode to see inferred type
// Then copy-paste it as explicit type

// Inferred: Promise<User | null>
async function getUser(id: string): Promise<User | null> {
  return this.userRepository.findOne({ where: { id } });
}
```

## Migration Checklist

- [ ] Add return types to all functions
- [ ] Replace `any` with specific types or `unknown`
- [ ] Replace `console.*` with Logger
- [ ] Change `let` to `const` where not reassigned
- [ ] Remove unused imports/variables
- [ ] Fix naming conventions (no I prefix, PascalCase for classes)
- [ ] Add accessibility modifiers
- [ ] Fix boolean expressions (explicit null checks)
- [ ] Replace `||` with `??` for defaults
- [ ] Add NestJS class suffixes

## Performance Tips

```bash
# Use cache (2-10x faster on subsequent runs)
npx eslint src --config eslint.config.extended.mjs --cache

# Lint only changed files
git diff --name-only | grep '\\.ts$' | xargs npx eslint --config eslint.config.extended.mjs

# Lint specific directory
npx eslint src/modules/user --config eslint.config.extended.mjs
```

## VSCode Integration

`.vscode/settings.json`:
```json
{
  "eslint.options": {
    "overrideConfigFile": "eslint.config.extended.mjs"
  },
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}
```
